How it works

01

Fetch. All sources hit in parallel — HN, arXiv, GitHub, HuggingFace, Reddit. Cached in a local SQLite DB at ~/.pi/radar/radar.db.

02

Deduplicate. Same article from multiple sources gets merged into one item. Extra source refs tracked automatically.

03

Pre-score. Items ranked by recency × signal strength in TypeScript before touching the LLM. Only the top 20 candidates go forward.

04

Rank. The LLM reads your ~/.pi/radar/profile.md and picks the 8 most relevant items. It only ranks — no fetching, no tool calls.

Commands

/radar digest Top 8 items from the last 24h, ranked for your profile
/radar trending Fastest-rising items by metric velocity
/radar explain <id> Deep dive — what it is, why it matters, 3 takeaways
/radar try <repo> Generates install/run steps for a GitHub repo
/radar mute <pattern> Silence a domain, author, source, or topic
/radar stats DB stats — item counts, sources, newest entry

Mute patterns: domain:medium.com  ·  author:someone  ·  source:reddit  ·  bare topic strings
